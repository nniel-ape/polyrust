# TailEnd Fast-Exit v2

## Overview

Redesign the TailEnd stop-loss exit mechanism for sub-second reaction to price reversals in 15-minute crypto markets.

**Problems solved:**
1. Exit triggers only fire on OrderbookUpdate events ‚Äî external price feeds (50-200ms ahead of CLOB) are ignored for exits
2. FOK exits reject entirely in thin markets, entering slow retry loops (2s cooldown, geometric clip reduction)
3. Sell delay (4s) gates all exits including hard crashes
4. Hedge (set completion) only triggers after 5 failed exit retries ‚Äî too late

**Key changes:**
- Add FAK (Fill-And-Kill) order type ‚Äî immediate partial fills for exits
- Evaluate exit triggers on ExternalPrice events (frontrun stale CLOB bids)
- Hard crash exits bypass sell delay (0s)
- Proactive hedge (opposite-side buy) simultaneously with exit
- Simplify lifecycle from 6 states to 4
- Timestamp correctness and source-priority fallback for fast risk decisions

## Technical Details

### New lifecycle state machine (4 states, replacing 6)

```
Current:  Healthy ‚Üí DeferredExit ‚Üí ExitExecuting ‚Üí ResidualRisk ‚Üí RecoveryProbe ‚Üí Cooldown
New:      Healthy ‚Üí ExitActive ‚Üí Hedged | (resolved)
                                        ‚Üí Cooldown (market-level only)
```

```rust
enum PositionLifecycleState {
    Healthy,
    ExitActive {
        sell_order_id: OrderId,
        sell_type: OrderType,          // Fak (initial) or Gtc (residual)
        sell_price: Decimal,
        submitted_at: DateTime<Utc>,
        filled_so_far: Decimal,
        hedge_order_id: Option<OrderId>,
        hedge_price: Option<Decimal>,
        trigger: StopLossTriggerKind,
    },
    Hedged {
        hedge_cost: Decimal,
        hedged_at: DateTime<Utc>,
    },
    Cooldown { until: DateTime<Utc> },  // market-level re-entry gating only
}
```

### FAK order type flow

```
trigger fires ‚Üí FAK sell at current_bid (depth-capped clip)
    ‚îú‚îÄ full fill ‚Üí position resolved
    ‚îú‚îÄ partial fill ‚Üí remaining tracked, GTC placed at bid-1tick
    ‚îÇ   ‚îú‚îÄ GTC fills ‚Üí position resolved
    ‚îÇ   ‚îî‚îÄ GTC stale (>2s) ‚Üí cancel, replace at new bid (chase)
    ‚îî‚îÄ zero fill ‚Üí GTC at bid-1tick (fallback)

simultaneously (if profitable):
    GTC buy opposite token at current_ask
    ‚îú‚îÄ fills ‚Üí Hedged state (set complete, $1.00 at expiry)
    ‚îî‚îÄ rejected ‚Üí continue sell-only
```

### Config changes

**New:**
- `fast_path_enabled: bool` (default: true)
- `fast_path_max_book_age_ms: i64` (default: 2000)

**Removed (backward-compat serde skip):**
- `max_exit_retries` ‚Äî no retry loop with FAK+GTC
