# TailEnd Fast-Exit v2

## Overview

Redesign the TailEnd stop-loss exit mechanism for sub-second reaction to price reversals in 15-minute crypto markets.

**Problems solved:**
1. Exit triggers only fire on OrderbookUpdate events ŌĆö external price feeds (50-200ms ahead of CLOB) are ignored for exits
2. FOK exits reject entirely in thin markets, entering slow retry loops (2s cooldown, geometric clip reduction)
3. Sell delay (4s) gates all exits including hard crashes
4. Hedge (set completion) only triggers after 5 failed exit retries ŌĆö too late

**Key changes:**
- Add FAK (Fill-And-Kill) order type ŌĆö immediate partial fills for exits
- Evaluate exit triggers on ExternalPrice events (frontrun stale CLOB bids)
- Hard crash exits bypass sell delay (0s)
- Proactive hedge (opposite-side buy) simultaneously with exit
- Simplify lifecycle from 6 states to 4
- Timestamp correctness and source-priority fallback for fast risk decisions

## Technical Details

### New lifecycle state machine (3 states, replacing 6)

```
Before:   Healthy ŌåÆ DeferredExit ŌåÆ ExitExecuting ŌåÆ ResidualRisk ŌåÆ RecoveryProbe ŌåÆ Cooldown
After:    Healthy ŌåÆ ExitExecuting ŌåÆ Hedged | (resolved)
```

```rust
enum PositionLifecycleState {
    Healthy,
    ExitExecuting {
        order_id: OrderId,
        order_type: OrderType,          // Fak (initial) or Gtc (residual)
        exit_price: Decimal,
        submitted_at: DateTime<Utc>,
        hedge_order_id: Option<OrderId>,
        hedge_price: Option<Decimal>,
    },
    Hedged {
        hedge_cost: Decimal,
        hedged_at: DateTime<Utc>,
    },
}
```

### FAK order type flow

```
trigger fires ŌåÆ FAK sell at current_bid (depth-capped clip)
    Ōö£ŌöĆ full fill ŌåÆ position resolved
    Ōö£ŌöĆ partial fill ŌåÆ remaining tracked, GTC placed at bid-1tick
    Ōöé   Ōö£ŌöĆ GTC fills ŌåÆ position resolved
    Ōöé   ŌööŌöĆ GTC stale (>2s) ŌåÆ cancel, replace at new bid (chase)
    ŌööŌöĆ zero fill ŌåÆ GTC at bid-1tick (fallback)

simultaneously (if profitable):
    GTC buy opposite token at current_ask
    Ōö£ŌöĆ fills ŌåÆ Hedged state (set complete, $1.00 at expiry)
    ŌööŌöĆ rejected ŌåÆ continue sell-only
```

### Config changes

**New:**
- `fast_path_enabled: bool` (default: true)
- `fast_path_max_book_age_ms: i64` (default: 2000)

**Removed (backward-compat serde skip):**
- `max_exit_retries` ŌĆö no retry loop with FAK+GTC
- `short_limit_refresh_secs` ŌĆö GTC chase is event-driven
- `short_limit_tick_offset` ŌĆö simplified to `gtc_fallback_tick_offset`
- `reentry_confirm_ticks` ŌĆö hedge replaces alpha re-entry
- `reentry_cooldown_secs` ŌĆö market-level cooldown only

**Kept:**
- `recovery_max_set_cost` (hedge profitability threshold)
- `exit_depth_cap_factor` (FAK initial clip sizing)
- `gtc_fallback_tick_offset` (GTC residual pricing)
- `gtc_stop_loss_max_age_secs` (stale GTC refresh)
